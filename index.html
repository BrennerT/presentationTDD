Die drei Regeln der testgetriebenen Entwicklung

1. You are not allowed to write any code unless it is to make a failing test pass

2. You are not allowed to write any more of a unit test than is sufficient to fail

=> As soon as test is red write production code

3. You are not allowed to write any more code than is sufficient to pass
